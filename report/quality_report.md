# Отчёт по анализу базы данных с кинофильмами

## 1. Введение

Данный датасет содержит информацию о 9463 кинофильмах, включая их названия, даты выхода, краткие описания, а также показатели популярности и рейтингов на основе оценок пользователей. Эти данные могут помочь с тем, чтобы лучше понять предпочтения людей, закономерности и зависимости между параметрами.

Предполагаемым заказчиком анализа может быть: Walt Disney Company; Базелевс; Yellow, Black and White Group; Paramount Pictures; Кинокомпания СТВ.

На основе этих данных могут быть решены следующие:
1. Выявить среднюю разницу оценки первой и последующих частей фильмов
2. Определить, какие слова чаще встречаются в описании фильмов с высокой оценкой
3. Найти наиболее удачные года кинопроката
4. Обнаружить закономерности дат выпуска относительно среднего успеха фильмов"

## 2. Паспорт датасета

Размер в формате (строки | столбцы) = (9463 | 8)
| Тип данных | Структурированные табличные данные |

### Ключевые признаки:

| Признак | Тип данных | Описание |
|---------|------------|----------|
| "id" | int64 | Уникальный идентификатор фильма |
| "title" | object | Название фильма |
| "release_date" | object | Дата выхода фильма в формате строки |
| "overview" | object | Краткое описание сюжета (может отсутствовать) |
| "popularity" | float64 | Числовой показатель популярности фильма |
| "vote_average" | float64 | Средняя оценка пользователей |
| "vote_count" | int64 | Количество оценок пользователей |
| "video" | bool | Наличие видео (вероятно, трейлера или ссылки) |

## 3. Основные проблемы качества данных

| № | Проблема | Описание | Пример/Причина |
|---|----------|----------|----------------|
| 1 | Пропуски в "overview" | У 14 фильмов отсутствует описание сюжета | "overview" = NaN |
| 2 | Некорректный формат "release_date" | "release_date" хранится как строка, возможно, с некорректными значениями | Строковый тип вместо datetime |
| 3 | Аномалии в "popularity" | Значения "popularity" имеют широкий разброс и могут содержать выбросы | Возможно, искусственно завышенные значения |
| 4 | Низкая надёжность "vote_average" | Средняя оценка может быть искажена при малом количестве голосов | "vote_average" = 10 при "vote_count" = 1 |
| 5 | Избыточность поля "video" | Все значения "video" одинаковы (в исходном датасете, а не в моём), что не даёт полезной информации | Нулевая дисперсия признака |

## 4. План улучшения качества

1. Преобразование "release_date"
   - Привести к типу "datetime"
   - Удалить или импутировать некорректные даты

2. Обработка пропусков в "overview"
   - Заменить NaN на "Нет описания"
   - Или удалить строки (если их мало)

3. Фильтрация по "vote_count"
   - Исключить фильмы с малым числом оценок (например, "vote_count" < 10)
   - Повышение достоверности "vote_average"

4. Нормализация "popularity"
   - Использовать scaling (MinMaxScaler) для приведения к единому диапазону

5. Удаление бесполезного признака "video"
   - Исключить столбец из анализа, если все значения идентичны

6. Создание новых признаков
   - Извлечь год выпуска из "release_date" для временного анализа
   - Рассчитать взвешенный рейтинг (например, по формуле Bayesian average)

7. Обработка выбросов
   - Для "popularity" и "vote_count" применить логарифмирование или Winsorization

8. Токенизация "overview"
   - Для текстового анализа извлечь ключевые слова или темы

## 5. Краткое резюме

В процессе анализа я научился выявлять ключевые проблемы качества данных в предметной области кино, такие как пропуски в текстовых полях, некорректные форматы дат и статистические аномалии в числовых признаках. Особое внимание следует уделить нормализации рейтингов с учётом количества голосов, так как это напрямую влияет на достоверность выводов. Также важно привести даты к стандартному формату для корректного временного анализа.

При дальнейших шагах ИАД критично сбалансировать данные (например, по годам выпуска или жанрам, если они появятся) и проверить гипотезы о взаимосвязи популярности и оценок. Текстовый анализ описаний ("overview") может стать мощным инструментом для кластеризации фильмов по тематике, но требует тщательной предобработки и очистки от стоп-слов.
